Tempo de compilacão e tempo de execução

São termos utilizados para distinguir dois momentos distintos na construção de um software. Tempo de compilação, ou compiling-time, é o momento em que o código-fonte é convertido em um código de máquina, enquanto o tempo de execução, ou runtime, é o momento em que o código de máquina está sendo executado.

Durante a faze de compilação o código fonte é parseado, tem sua sintaxe, semântica e bibliotecas verificadas, para então criar um executável, que pode ser um código de máquina, como saída. Enquanto na fase de execução, ou running time, um novo processo é criado com o código de saída (ou seja, um código binário, ou um código de máquina, etc.) Durante a execução, depois da compilação, os erros produzidos são inesperados, portanto, não são detectados pelo compilador.

Exemplo de erro de compilação:

  
  #include <iostream>

  int main() {
    int number_one = 1;
    std::cout << number_one
  }

  #error: expected ';' after expression

Exemplo de erro de execução:

  
  #include <iostream>

  int main() {
   int number_one  = 1;
   int number_zero = 0;
   std::cout << number_zero / number_zero;
  }

  #error: Floating point exception
