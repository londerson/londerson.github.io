Compile time and runtime

Two terms used to distinguish different moments in building software. Compile-time is a moment at which the source code is converted into executable code. And runtime is a moment at which this executable-code it’s running.

Compiling is the process where the source code is parsed, checked semantics, syntax, linked libraries, then creating an executable - may be machine code - as output.

Somethings can go wrong in compile-time like time-checking errors, syntax errors, etc.

Running is the new process where the binary-code, or machine-code, is executed on the computer. During the execution, after compilation, errors are produced as an unexpected result don’t detected by compiler before.

Compile error - example:

  
  #include <iostream>

  int main() {
    int number_one = 1;
    std::cout << number_one
  }

  #error: expected ';' after expression

Runtime error - example:

  
  #include <iostream>

  int main() {
   int number_one  = 1;
   int number_zero = 0;
   std::cout << number_zero / number_zero;
  }

  #error: Floating point exception
